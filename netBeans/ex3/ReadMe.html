Some points - Futures

Future - Cookies

 In our project, we use a clever system of working with cookies.
In what appears the system:
  • First, the user can only vote once for each link.
  • Second user does not have to vote once for all links, that is, he can vote for as many options as he wants and it will next be able to vote for those for which he has not voted but it will not be able to vote for those for which he had voted before.
  • Thirdly there is also use cookies to count the number of users, that is, if a person has voted at least once, he scored once, if he will vote again this is not counted towards!

Cookies will be stored for one month. It can be change in voting servlet.

Future - Cookies/ Output Errors

 Because of the technical features of the cookies as well as the specific approach this problem in our group.
Which is manifested in the construction of forms of voting, which in turn is built dynamically for each user, depending on its previous votes.
Therefore, in this case, we are also using cookies to store error numbers.
Which can be two options:
The first is the introduction of a null value or a letter
and the second error is when the value entered is not suitable for a possible value.

At the same Advantages to face:
  • We do not use java script
  • The user who wants to vote two or more times for the same thing by pressing [F5] will not be able to do so until an entry in the cookies.

Future - Working with the input file

 Also our project can detect the absence of a file (file not exist) and show the error.
But other than that the main feature is a file verification check on the occurrence of values in the file!
That is, if the file suddenly for some reason was empty, then the error about it will be displayed that shows the input file is empty!
